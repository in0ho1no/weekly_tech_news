# AIは走り続けている

AI関連の変化はすさまじいスピードで続いています。  
GitHub Copilot周辺のコミュニティを見ると、その中の人たちでさえ「最新情報のすべてには追いつけない」という声が上がるほどです。  

本記事では、AI開発が止まりにくいほど加速している現実と、導入しても成果が自動的には出ない現実の両方を踏まえ、私たちが今の段階で何を準備できるのかを考えます。

## 開発を止めるべきだという声

### Anthropicの提言

2026年6月第1週、Claude等のAIを開発するAnthropicが、AI開発に関する提言を発表しました。  
今すぐ止めろということではなく、「必要な場合に開発を止められる仕組みを、複数のAI企業と各国政府が協調して整えるべきだ」という内容です。

:::source
Anthropic「When AI builds itself」
<https://www.anthropic.com/institute/recursive-self-improvement>
:::

Anthropicが公開した内部データによると、同社のコードベースに取り込まれたコードの80%以上はすでにAIが書いたものだそうです。  
また、AIが人間の介入なしに自律して完了できるタスクの規模（時間軸）は約4ヶ月ごとに倍増しています。2024年初頭には「人間の作業時間で4分程度」の短いタスクしかこなせなかったAIが、2026年には「12時間規模」の連続した複雑なタスクをこなすようになっているとのことです。

技術の進歩が競争優位に直結する状況では、単純に「少しだけペースを落とす」という中間策は取りにくくなります。こうなると、現実には「完全に止める」か「走り続ける」かの二択に近づいてしまいます。  

### 過去にも同様の提言はあった

2023年には、イーロン・マスク氏やスティーブ・ウォズニアック氏など1000人以上が「高度なAIの開発を6ヶ月間停止すべき」という公開書簡に署名しました。  
結果として、その時は開発が止まるような気配は感じられませんでした。  

:::source
「AI開発6カ月停止を」イーロン・マスク氏ら署名の公開書簡
<https://gigazine.net/news/20230330-pause-ai-training/>
:::

3年前よりも著しくAI関連技術が発展している現在では分かりませんが、同様に相手にされない可能性は十分考えられます。

## 使っても成果が出ないという現実も

積極的な活用事例がある一方で、活用を進めても成果に繋がらない事例も出てきています。  

配車サービスのUberでは、エンジニアの95%がAIツールを日常的に利用しており、コミットされるコードの70%はAIが生成したものだそうです。  
同社のCTOが「2026年分のAI予算をわずか4ヶ月で使い果たしてしまった」と明かしたことで社内に激震が走りましたが、さらに同社COOも、その莫大なトークン支出がユーザー向けの機能向上にどう繋がったのか明確な因果関係を示せず、投資に見合う成果を正当化するのが難しくなっていると指摘しています。

:::source
Uber COO Andrew Macdonald: AI token spending harder to justify
<https://www.businessinsider.com/uber-coo-andrew-macdonald-ai-token-spending-harder-justify-2026-5>
:::

導入したからといって、自動的に成果に繋がるわけではない。  
この事例は、先行企業でさえ試行錯誤している段階であることを示しています。

## それでも、準備はできる

立場や置かれた環境によって判断は分かれますし、その前提自体も急速に変化しています。  
だからこそ、AIを本格的に活用する段階になったときに備えて、今のうちから使いどころや向き合い方を整理しておくことには意味があるはずです。
