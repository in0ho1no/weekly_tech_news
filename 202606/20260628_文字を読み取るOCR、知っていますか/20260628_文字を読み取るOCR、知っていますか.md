# 文字を読み取るOCR、知っていますか

## きっかけ

2026年6月23日、フランスのAI企業Mistral AIが文書解析モデル「Mistral OCR 4」を公開しました。  
PDF・Word・PowerPointなどの文書からテキストや表を高精度に抽出するモデルで、日本語を含む170言語に対応しています。  

このニュースを見て、「そもそもOCRって何だろう」と思った方もいるかもしれません。  
今回はOCRの基本とAI活用で必要になる場面を整理します。  
実際に試すことよりも、まずは「どんな技術で、なぜ必要か」を理解することを目的にします。

## OCRとは

OCRは「Optical Character Recognition」の略で、日本語では「光学文字認識」と訳されます。  
簡単に言うと、画像やスキャンデータの中に写っている文字を、
コンピュータが扱えるテキストデータとして読み取る技術です。  

身近な例を挙げると、以下のような場面で使われています。  

- 紙の書類をスキャンして、テキストとして保存する
- 手書きの伝票や申請書をデジタル化する
- 画像として保存されているPDFから文字を取り出す

:::note
PDFには大きく2種類あります。
文字として保存されているPDFと、画像として保存されているPDFです。
Word等で作ったPDFは前者で、紙をスキャンして作ったPDFは後者に該当することが多いです。
後者はそのままではテキストとして取り出せないため、OCRが必要になります。
:::

## なぜ改めてOCRが話題になるのか

AIツールは、与えた情報をもとに回答を作ります。  
逆に言えば、情報として渡してあげなければ、十分な回答を得られないことになります。  

「紙の資料」や「スキャンした画像PDF」しかないとき、AIにその内容を読ませるためには、まずテキストとして取り出す必要があります。  
OCRはその入り口になります。  

先週の記事で触れたMarkdownも同様です。  
AIに渡しやすい形でドキュメントを整備しようとするとき、元の資料が画像PDFや紙のままでは始まりません。  
OCRでテキスト化して、はじめてMarkdownへの変換や整理が可能になります。  

:::info
テキスト化できていない資料は、AIにとって「存在しない情報」と同じと言ってもいいでしょう。
活用の第一歩は、資料を「読める形」にすることから始まります。
:::

:::note
最近のAIはテキスト以外も読んでくれたり、ファイル読み取りのためのスクリプトを自ら書いてくれることもあります。
しかし、都度スクリプトを書かせたり、AIにテキスト以外の情報を直接読ませるのはコストがかかります。
自前でテキスト化する手段を知っておくと、今後の役に立ちます。  
:::

## AIで性能が上がっている

Mistral OCR 4の例に戻ると、このモデルはOCR専用に作られたAIモデルです。  
単に文字を読み取るだけでなく、以下のような情報を合わせて出力できます。  

- 文字や図がページのどこにあるか
- 見出し・表・数式・署名など、要素の種類の分類
- どれくらい正確に読み取れたかを示す信頼度スコア

「文書が何を言っているか」だけでなく、「どこに何がある」「どの程度確かな読み取りか」まで
構造化して出せるようになったのは、AI技術の進歩によるものです。  

:::caution
Mistral OCR 4はAPI経由の有料サービスです。
個人や業務での利用前に、料金体系とルールを必ず確認してください。
:::

## 知っておきたい無料OCRツール

Tesseractという定番もありますが、今回は国内で公開されている無料のOCRツールを、知識として押さえておきましょう。

国立国会図書館は2026年2月、「NDLOCR-Lite」を公開しました。  
国立国会図書館がこれまで開発してきたOCRの軽量版で、以下の特徴があります。  

- 無料で利用できる
- GPUなど専門的なハードウェアが不要
- デスクトップアプリとして提供されており、マウス操作だけで使える
- Windows・Mac・Linuxに対応

図書や雑誌のデジタル化画像からテキストを取り出すことを想定して作られており、英文や手書き文字にも実験的に対応しています。  

:::caution
NDLOCR-Liteはオープンソースで公開されているツールです。
業務資料への利用は各所属のルールに従ってください。
私からの紹介は個人の学習用途で、OCRの仕組みを理解するための利用が主な想定です。
:::

## 今日から意識できること

OCRは専門家だけが使うツールではなく、「紙やスキャンPDFをデジタル化する」という場面で誰もが関わりうる技術です。

紙資料、画像として保存されているPDFなどをAIで活用しようとするとき、まず必要になるのがこのOCRという工程です。

まずは「そのPDFは文字データか、画像データか」を見分ける視点だけでも持っておくと、実務で迷いにくくなります。  
AIへの情報の渡し方を考えるときは、作業を増やしすぎないためにも「入り口」の整備を意識してみるといいでしょう。
